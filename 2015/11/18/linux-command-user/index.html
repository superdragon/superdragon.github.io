linux用户、用户组、权限

发表于 2015-11-18   |   分类于 Linux   |  

用户组分类

  1. 管理员root: 具有使用系统所有权限的用户,其UID 为0.
  2. 普通用户: 即一般用户,其使用系统的权限受限,其UID为500-60000之间.
  3. 系统用户: 保障系统运行的用户,一般不提供密码登录系统,其UID为1-499之间.

与用户有关的文件/etc/passwd, /etc/shadow

/etc/passwd 保存账户的基本信息

每个位置的意义：

  1. 账号
  2. 密码占位符(密码存放在/etc/shadow)
  3. 用户ID号 UID
  4. 用户组ID号 GID
  5. 用户详细信息
  6. 用户home目录
  7. 编译环境

/etc/shadow 存放用户密码

新建用户

useradd 用户名 用户组默认和用户名相同，自动在home下面创建test3用户目录。

useradd -g 用户组 用户名 添加账户并归属到某用户组，此用户组必须已存在。

passwd 用户名 修改或者初始化用户密码，如果设置的密码过于简单，系统会提示你，但仍然可用。

删除用户

userdel [-r] 用户名 删除账户，如果加-r，会同时删除用户的home目录。

修改用户

usermod -l 新用户名 原用户名 修改用户名称，但home下的目录名不修改，如需修改，可以用 mv 命令修改。

查询用户

id 用户名

用户权限

文件或目录的访问权限分为只读，只写和可执行三种。分别是r，w，x。数字代替是4，2，1。如果非自己账户登录时，想访问自己的目录，那么必须是 x 权限才行。

每个文件或者目录都用三组不同类型的用户访问权限，分别是自己，同组用户，其他用户。

每个文件或目录的权限都有10个‘-’标示。

  1. 第1个：代表是目录还是文件，d=目录，-=文件
  2. 第2-4个：代表自己拥有的权限。
  3. 第5-7个：代表同组拥有的权限。
  4. 第8-10个：代表其他拥有的权限。

例如上面的例子：只有test1用户能够访问并读写此目录，同组和其他用户无任何操作权限。

修改权限

chmod [-R] 目录或文件名称 修改某个目录或者文件的权限，加 -R 则会一起修改子目录及子文件夹的权限。

例如：用test1账户登录，在/home/test1目录下创建test目录，在/home/test1/test目录下创建test.txt文件。

  1. test.txt文件，自己和同组用户都有可读可写权限，其他用户只有可读权限。
  2. test目录，自己和同组用户都有所有权限，其他用户只有访问权限。

test目录，自己有所有权限，同组用户和其他用户只有访问和可读权限。在没有加[-R]可选项时，test目录中的text.txt子文件的权限并没有变化。

test目录中的text.txt，自己有所有权限，同组用户和其他用户有可读可写权限。
